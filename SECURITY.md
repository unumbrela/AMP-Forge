# Security Policy

## Supported Scope

Security reports are welcome for:

- Repository code and workflow configuration.
- Dependency-related vulnerabilities affecting this project.
- Sensitive data exposure risks in tracked files.

## Reporting a Vulnerability

Please do not open public issues for active vulnerabilities.

Instead, report privately with:

- affected file/path,
- impact and exploitation scenario,
- steps to reproduce,
- suggested mitigation (if available).

## Response Expectations

- Initial acknowledgement: within 5 business days.
- Triage and severity assessment: as soon as reproducible details are provided.
- Fix timeline: depends on severity and blast radius.

## Safe Harbor

Good-faith vulnerability research and responsible disclosure will be respected.
